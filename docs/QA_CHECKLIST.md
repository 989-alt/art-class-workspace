# Art Class Workspace v3 — 수동 QA 체크리스트

v3 LMS 출시 게이트(`art-class-v3-lms-plan.md`)에 대응하는 수동 E2E 시나리오. 자동화 테스트 러너는 도입하지 않으므로 배포 직전 사람이 직접 따라가며 통과시킨다.

권장 환경:
- 교사 PC: Chrome 최신 (1920×1080 + 보조 디스플레이 또는 TV HDMI)
- 학생 기기: 시크릿 창 3개 + 모바일 Safari/Chrome 1대 이상
- 네트워크: 학교 Wi-Fi 시뮬레이션 (DevTools Network → Fast 3G)

---

- [ ] `.env.local` 에 `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` 설정
- [ ] Supabase SQL Editor에서 마이그레이션 순서대로 실행

  - `classroom-assets` (5MB · `image/png, image/jpeg, image/svg+xml`)
  - `classroom-submissions` (5MB · `image/png, image/jpeg`)
- [ ] Supabase Authentication에서 교사 계정 1개 생성 (Auto Confirm 권장)
- [ ] Gemini API 키 발급 (https://aistudio.google.com/apikey)
- [ ] `npm install && npm run build && npm run preview` 로 dist 서빙 확인

## 1. 교사 단독 동선

### 1-1. 학급 생성

- [ ] 앱 접속 → API 키 입력 → 메인 워크스페이스 진입
- [ ] 헤더의 "🏫 내 학급" 클릭
- [ ] 미로그인 상태면 교사 로그인 화면(TeacherAuthGate) 노출 → 0단계의 교사 계정으로 로그인
- [ ] 학급 미생성 안내 문구 + "새 학급 만들기" 폼 표시
- [ ] 학급 이름 입력 → 생성 버튼 → 6자리 코드 + QR + 학급 URL이 영구 노출 확인
- [ ] 페이지 새로고침 → 동일 코드/QR이 그대로 유지

### 1-2. 도안 생성 → 학급에 게시

- [ ] "교과서" 탭에서 "3-1 미술 우리 동네의 모습" 프리셋 선택 → 세부 주제 칩 1개 선택 → 도안 생성
- [ ] 결과 패널의 **"📤 우리 학급에 게시"** 버튼 활성 확인
- [ ] 클릭 → 과제 제목 입력 모달 → 제출
- [ ] "게시 성공" 토스트 확인
- [ ] "내 학급 보기" → 과제 리스트에 새 과제 즉시 표시 (썸네일 + 제목)
- [ ] 자유 모드 / 만다라 모드에서도 "📤 우리 학급에 게시" 동일하게 동작

### 1-3. 학급 관리

- [ ] 학급 이름을 다른 이름으로 변경 → 저장 → 새로고침 후 새 이름 유지
- [ ] 과제를 1개 추가 게시해 리스트가 누적됨을 확인 (가장 최근이 위)

## 2. 학생 동선 (다른 브라우저 / 시크릿)

### 2-1. 입장

- [ ] 시크릿 창에서 교사 화면의 QR 스캔 또는 `#/class/ABCDEF` 직접 입력
- [ ] 닉네임 입력 폼 → "이상한고래" 등 입력 → 입장
- [ ] 학급 과제 리스트 화면 진입
- [ ] 페이지 새로고침 → 닉네임이 자동 복원되어 닉네임 입력 폼이 다시 뜨지 않음
- [ ] 닉네임 변경 UI(있다면)로 다른 닉네임으로 변경 → 새로고침 후 변경된 닉네임이 유지

### 2-2. 과제 다운로드

- [ ] 과제 카드 클릭 → 도안 프리뷰 화면 진입
- [ ] "📥 PNG 다운로드" 버튼 클릭 → 파일이 정상 저장됨
- [ ] 파일을 열어 도안이 깨짐 없이 표시되는지 확인

### 2-3. 작품 제출

- [ ] "📸 사진 선택" → 색칠본 사진(또는 임의 PNG) 업로드
- [ ] "✅ 제출 완료! 검수 대기중" 카드 표시
- [ ] 새로고침 시에도 본인의 제출 상태가 유지되는지 확인 (브라우저 로컬 상태)
- [ ] 다른 시크릿 창(다른 학생 시뮬)에서는 본인 제출이 보이지 않음 (미승인은 비공개 RLS 검증)

---

## 3. 다중 학생 시뮬

시크릿 탭 3개 + 모바일 1대로 동시에 진행.

- [ ] 4개 클라이언트 각각 다른 닉네임 + 다른 사진 제출
- [ ] 교사 화면의 검수 패널에 4건 모두 표시되는지 확인
- [ ] Realtime 구독으로 새 제출이 ≤ 2초 내에 교사 화면에 반영되는지 확인 (수동 새로고침 없이)
- [ ] 같은 시크릿 창(같은 `student_token`)에서 다시 제출하면 새로운 pending 행이 생성됨 → 교사는 가장 최근 제출 1건만 승인하면 됨

## 4. 교사 검수 / 전시장

### 4-1. 검수 패널

- [ ] 교사 화면의 "내 학급 → 과제 → 제출" 영역에 미승인 썸네일이 ⏸ 배지로 표시
- [ ] 썸네일 클릭(또는 토글) → ⏸ → ✅
- [ ] 다시 클릭 → ✅ → ⏸ (해제 가능)
- [ ] 닉네임이 캡션으로 표시됨

### 4-2. 학급 전시장

- [ ] "🎨 학급 전시장 열기" → 전시장 화면
- [ ] 승인된 작품만 그리드에 표시 (미승인은 노출되지 않음)
- [ ] 미승인을 추가 승인 → 전시장에 즉시 추가
- [ ] 닉네임 캡션 표시 확인

### 4-3. TV 모드

- [ ] "📺 TV 모드" 클릭 → 풀스크린 슬라이드쇼 진입 (브라우저 fullscreen 허용)
- [ ] 자동 슬라이드 (수 초 간격) 동작
- [ ] Space → 다음 슬라이드, → 다음, ← 이전, Esc → 종료
- [ ] Esc 종료 시 TV 버튼으로 포커스 복귀 (WCAG 다이얼로그 패턴)

### 4-4. 카탈로그 PDF

- [ ] "📄 카탈로그 PDF" 클릭 → 다운로드
- [ ] PDF를 열어 표지(학급 이름·날짜·과제 컨텍스트) 확인
- [ ] 4컷 그리드 페이지 + 각 작품 닉네임 캡션 확인
- [ ] **알려진 제약**: jsPDF 기본 빌드는 한국어 폰트 미임베드라 표지·캡션의 한글이 tofu(□)로 보일 수 있음. v3.1 후속 과제로 별도 트래킹.

## 5. 에러 경로

### 5-1. Supabase 미설정

- [ ] `.env.local` 의 `VITE_SUPABASE_URL` 을 비우고 재빌드
- [ ] BYOK 도안 생성, SVG, PDF 분할 출력은 정상 동작
- [ ] 헤더의 "🏫 내 학급" 클릭 시 "Supabase 설정이 필요합니다" 안내가 노출되며 앱이 죽지 않음
- [ ] 학생 경로 `#/class/ABCDEF` 진입 시 "이 기능은 사용할 수 없습니다" 카드 표시

### 5-2. 잘못된 학급 코드

- [ ] `#/class/AAAAAA` (존재하지 않는 코드) → "학급을 찾을 수 없어요" + 처음으로 돌아가기 버튼
- [ ] 5자리 등 잘못된 길이 → 코드 형식 오류 안내

### 5-3. Storage 버킷 미생성

- [ ] 일부러 `classroom-assets` 버킷을 삭제하고 "📤 우리 학급에 게시" 시도
- [ ] 친화적 에러 메시지(예: "도안 업로드에 실패했습니다. 관리자에게 문의하세요") 표시 + 앱 정상 유지
- [ ] 동일하게 `classroom-submissions` 가 없을 때 학생 제출도 친화적 에러

### 5-4. 네트워크 끊김

- [ ] 학생 제출 직전 DevTools → Network → Offline → 제출 클릭 → 실패 메시지
- [ ] 온라인 복귀 후 재시도 가능
- [ ] 교사 Realtime 구독 끊김 → 자동 재연결 (Supabase 기본)

### 5-5. API 키 만료 / 잘못됨

- [ ] 무효한 Gemini 키 입력 → "유효하지 않은 API 키" 에러 토스트, 앱 정상 유지
- [ ] 헤더 ⚙️ → "키 삭제 및 로그아웃" → 첫 진입 화면으로 돌아감

## 6. 접근성 / 성능 (수동 Lighthouse)

- [ ] BYOK 단독 첫 진입 페이지에서 Lighthouse 데스크톱 실행
  - [ ] 접근성 ≥ 95
  - [ ] 성능 ≥ 90 (목표, 메인 번들 크기에 따라 변동)
- [ ] 학생 페이지(`#/class/:code`)를 모바일 뷰포트(375×667)로 열어 반응형 확인
- [ ] Tab 키만으로 메인 → 학급 패널 → 도안 생성 → 결과 → 학급 게시까지 도달
- [ ] TV 모드 슬라이드쇼: ←/→/Space/Esc 모두 동작
- [ ] 모든 `<img>`에 의미 있는 `alt` 또는 `aria-hidden`
- [ ] Toast: success/info → `role="status"`, error → `role="alert"`

## 7. 영구 보관 검증

- [ ] 학급 만들고 24시간+ 후 (또는 다음날) 재방문
  - [ ] 학급 코드 동일
  - [ ] 과제 리스트 그대로
  - [ ] 승인된 학생 작품이 전시장에 그대로 남아 있음
- [ ] Supabase 대시보드에서 `expires_at` 같은 만료 컬럼이 어떤 테이블에도 없음
- [ ] `cleanup_expired_*` 함수가 데이터베이스에 존재하지 않음 (제거 검증)

## 8. 회귀 (v2와의 차이)

v2 잔재가 UI에 남아 있지 않은지 확인.

- [ ] **투표 UI 없음** — "키워드 투표", "선 굵기 투표", "학생 투표 시작" 같은 문구가 어디에도 노출되지 않는다
- [ ] **워터마크 토글 없음** — ExportPanel에 워터마크 On/Off 옵션이 없다
- [ ] **저작권 보증서 토글 없음** — ExportPanel에 교사 이름 입력란·보증서 On/Off 옵션이 없다
- [ ] **색약 시뮬 탭 없음** — CanvasPreview 상단에 정상/D/P/T 탭이 없다 (선 굵기 +20% 수동 버튼만 남음)
- [ ] **24h 자동 삭제 멘트 없음** — Storage·정책 어디에도 24시간 만료 안내가 없다
- [ ] v1 핵심 회귀 없음
  - [ ] 자유 모드 "사과 바구니" 생성 정상
  - [ ] 만다라 모드 정상
