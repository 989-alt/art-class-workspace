# AI Art Class Workspace — 수동 QA 체크리스트

Phase 3 출시 게이트(Task 9)에 대응하는 **수동 E2E 시나리오**입니다. 자동화 테스트 러너(Playwright/Vitest)는 도입하지 않으므로, 배포 직전 사람이 직접 따라가며 각 항목을 통과시킵니다.

권장 실행 환경:
- 교사 PC: Chrome 최신 (1920×1080 + 보조 디스플레이 or TV HDMI)
- 학생 기기: iOS Safari / Android Chrome 각 1대 이상 + 시뮬레이션용 데스크톱 시크릿 창 3~4개
- 네트워크: 학교 Wi-Fi 시뮬레이션 (스로틀링 Fast 3G로 검증)

---

## 0. 사전 준비 (Pre-flight)

- [ ] `.env`에 `VITE_SUPABASE_URL` · `VITE_SUPABASE_ANON_KEY` 설정
- [ ] Supabase SQL Editor에서 `supabase/migrations/0001_classroom_schema.sql` ~ `0007_class_gallery_read.sql`을 **순서대로** 실행
- [ ] Supabase Storage에 **public** 버킷 `session-submissions` 생성 (Policy는 마이그레이션에서 이미 설정)
- [ ] Gemini API 키 발급 (https://aistudio.google.com/apikey) — 사용량 한도 확인
- [ ] `npm install && npm run build` 성공
- [ ] `dist/`를 정적 서빙(`npm run preview`)하거나 Vercel/GitHub Pages에 배포

## 1. 교사 혼자 — BYOK 단독 흐름 (v1 회귀)

학급 모드·Supabase 없이도 v1 기능이 그대로 동작하는지 확인합니다.

### 1-1. 온보딩
- [ ] 앱 접속 → API 키 입력 화면이 먼저 뜸
- [ ] 잘못된 키 입력 → "유효하지 않은 API 키입니다" 에러
- [ ] 올바른 키 입력 → 메인 화면 진입
- [ ] 헤더 우측에 `🔑 sk-****XYZ` 마스킹 확인
- [ ] ⚙️ 클릭 → API 키 모달 → "키 삭제 및 로그아웃" 정상 동작

### 1-2. 교과서 모드 1-클릭 도안
- [ ] "교과서" 탭 선택 → 10개 프리셋 카드 표시
- [ ] "3-1 미술 우리 동네의 모습" 카드 클릭 → 세부 주제 5개 칩 표시
- [ ] "동네 놀이터" 칩 선택
- [ ] 난이도/방향/용지/그리드가 프리셋 기본값으로 자동 세팅됨
- [ ] "🎨 도안 생성" 클릭 → 스켈레톤 10~15초 → 도안 표시
- [ ] 도안 우측 하단 CVD 탭 (정상·D·P·T) 토글 정상 동작
- [ ] 선 굵기 +20% 적용 → "이미 적용됨" 안내 확인
- [ ] Undo → 보정 해제 + 재적용 가능

### 1-3. 내보내기 + 보증서
- [ ] 사이드바 "내보내기" 패널에서 교사 이름 입력
- [ ] 워터마크 On, 보증서 On 상태에서 "📄 PDF N×M" 클릭
- [ ] 생성된 PDF 확인:
  - [ ] 각 페이지 하단에 `SEONBI's Art Class · Gemini · YYYY-MM-DD HH:mm` 워터마크
  - [ ] 마지막 페이지에 보증서 (단원명·프롬프트·SHA-256·QR)
  - [ ] QR 스캔 → 진위 확인 URL (해시 포함)
- [ ] 워터마크 Off → 다시 출력 → 워터마크 없음 확인
- [ ] 보증서 Off → PDF 페이지 수가 1장 줄어듦
- [ ] "🖼️ PNG 원본" / "📐 SVG 벡터" 각각 다운로드 정상

### 1-4. 갤러리 + ZIP
- [ ] 생성 횟수를 3으로 올려 3장 생성 → 갤러리 뷰로 자동 전환
- [ ] 카드 2장 체크박스 선택 → "📥 선택 다운로드 (2)" → ZIP 파일에 PNG 2장
- [ ] "📦 전체 다운로드" → 3장 전부 포함된 ZIP
- [ ] 썸네일 클릭 → 상세 뷰 이동 → "← 갤러리로 돌아가기" 복귀

## 2. 교사 + 학생 20명 학급 모드 (Phase 2/3)

### 2-1. 세션 생성 & QR
- [ ] "교과서" 탭에서 프리셋 선택 → 사이드바에 "📡 학급 모드로 진행" 버튼이 Supabase 설정된 경우에만 활성
- [ ] 버튼 클릭 → 교사 로그인 화면
- [ ] 처음이라면 회원가입 → 이메일 인증 후 로그인 (Supabase 정책 의존)
- [ ] 로그인 후 "학급 세션 준비" 화면 → 투표 후보 편집
- [ ] "📡 학급 세션 시작" 클릭 → **1초 이내** 6자리 코드 + QR 표시
- [ ] 코드는 I/O/0/1이 제외되었는지 확인 (혼동 방지)
- [ ] `window.location.origin/session/XXXXXX` URL 표시 확인

### 2-2. 학생 웹뷰 (시뮬레이션)
- [ ] 시크릿 창 4개 + 모바일 2대 → 각각 QR 스캔 or URL 직접 입력
- [ ] `/session/ABC123` 진입 → Supabase 미설정 시 "기능 사용 불가" 카드 (차단 환경 테스트)
- [ ] 정상 접속 시: 단원 라벨 + 키워드·선 굵기·디테일 3개 radiogroup
- [ ] 닉네임 미입력 상태로 "제출하기" → `학생🐼` 같은 이모지 닉네임 자동 생성
- [ ] 세 가지 중 하나라도 미선택 → "세 가지 모두 선택해주세요" 경고
- [ ] 제출 완료 → "제출 완료! 선생님이 투표를 마감할 때까지 기다려주세요" 카드 + 에코

### 2-3. 실시간 집계
- [ ] 교사 화면 `VoteDashboard`에 **2초 이내** 투표 반영
- [ ] 키워드 워드클라우드/막대그래프 실시간 업데이트
- [ ] 같은 토큰(같은 시크릿창)에서 2번 제출 → 2번째는 거부 (UNIQUE 제약)
- [ ] 시크릿창을 닫았다 다시 열면 새 토큰 → 다른 학생으로 집계

### 2-4. 투표 마감 → 도안 생성
- [ ] 20개 투표(또는 그 이상) 쌓인 상태에서 "🔒 투표 마감 + 도안 생성" 클릭
- [ ] 상태 pill이 `투표 진행 중` → `도안 생성 중`으로 전환
- [ ] Gemini 호출 실패 시: 자동 `reopenSession()`으로 `voting`으로 롤백, Toast 에러 표시
- [ ] 성공 시: `완료됨` pill + 완성본 카드 + 최종 프롬프트 `<details>`
- [ ] 동일 타이밍에 학생 화면이 "완성!" 화면으로 자동 전환되며 `PNG 다운로드` 버튼 동작

### 2-5. 레이스 조건
- [ ] 교사가 "투표 마감"을 누르는 찰나에 학생이 새 제출 시도 → 학생은 "세션이 종료되었습니다" 카드 확인
- [ ] 교사가 투표 마감 도중 네트워크 끊김(브라우저 DevTools 오프라인) → 에러 Toast, 세션은 `voting`으로 롤백

## 4. 에러 경로 (Negative Path)

### 4-1. Supabase 미설정
- [ ] `.env`에서 Supabase URL 제거 → 앱 재빌드
- [ ] GeneratorForm의 "📡 학급 모드로 진행" 버튼이 disabled + "학급 모드는 Supabase 설정이 필요합니다" 툴팁
- [ ] 학생 경로 `/session/ABC123` 접속 → "이 기능은 현재 사용할 수 없습니다" 에러 카드 (Graceful degradation)
- [ ] BYOK 단독 기능(교과서 모드·PDF·보증서)은 그대로 동작

### 4-2. 잘못된 세션 코드
- [ ] `/session/aaaaaa` (소문자 6자리) → 대문자로 정규화되어 실제 조회 시도
- [ ] `/session/ZZZZZZ` 같은 존재하지 않는 코드 → "세션을 찾을 수 없어요" + 뒤로가기 버튼
- [ ] `/session/12345` (5자리) → "잘못된 코드입니다" (regex 검증 실패)

### 4-3. 네트워크 중단
- [ ] 학생이 투표 제출 직전 오프라인 → submitVote 에러, 화면에 메시지
- [ ] 온라인 복귀 → 재시도 가능
- [ ] 교사 Realtime 구독 끊김 → 재연결 후 자동 동기화 (Supabase 기본 동작)

### 4-4. 안전 필터
- [ ] 자유 모드에서 정책 위반 키워드 입력 → "⚠️ 안전 정책 위반" Toast + 크레딧 소모 없음 확인
- [ ] 만다라 모드는 자유 입력 차단되어 있어 해당 없음

## 5. 접근성 (WCAG 2.1 AA)

### 5-1. 키보드 내비게이션
- [ ] Tab만 사용해 메인 화면 → 생성 → 갤러리 → 상세 → Quick Edit → Undo까지 도달 가능
- [ ] 갤러리 카드: Tab으로 체크박스 포커스, Space로 선택 토글, Tab 한 번 더 → 썸네일 버튼 → Enter로 상세 진입
- [ ] 학급 모드 투표 UI: radiogroup 내 화살표로 이동, Space로 선택
- [ ] TV 슬라이드쇼: Esc/←/→/Space 전부 반응

### 5-2. 스크린리더 (NVDA 또는 VoiceOver)
- [ ] 모든 이미지에 의미 있는 alt 또는 aria-hidden
- [ ] Toast 알림이 자동으로 읽힘 (success/warning: polite, error: assertive)
- [ ] 슬라이드쇼 열림·닫힘이 dialog로 안내됨
- [ ] 헤더 ⚙️ 버튼이 "API 키 설정"으로 안내됨

### 5-3. 색상 대비
- [ ] 크롬 DevTools Lighthouse 접근성 탭 실행 → 점수 ≥ 95
- [ ] 모든 텍스트가 WCAG AA 대비(일반 4.5:1, 큰 글씨 3:1) 통과
- [ ] 색약 시뮬 프리뷰에서도 선화가 식별 가능 (배경-선 대비 충분)

## 7. 24시간 자동 삭제

- [ ] Supabase SQL Editor에서 다음 실행:
  ```sql
  select cleanup_expired_sessions();
  ```
  현재 시점 기준 `expires_at < now()`인 세션이 삭제되고 cascade로 votes·submissions 함께 정리
